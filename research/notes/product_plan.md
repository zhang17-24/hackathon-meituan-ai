# NailFlow 产品方案草案

## 赛题切入

题目是“美甲 AI 试戴与智能运营”。建议把作品包装成一个面向美团美甲商家和消费者的双端 Agent：

- 用户端：上传手图，选择款式，生成保真试戴图，输出风格解释和可预约门店。
- 运营端：自动识别爆款趋势、生成运营日报、推荐上新/活动/客服话术。
- 评测侧：完整性、创新性、应用效果、商业价值都能被覆盖。

## 命题一：还原手部细节和 P 美甲更好

核心流程：

1. 输入手图和款式图。
2. MediaPipe/Hand Landmark 估计手部姿态，找到指尖、指节、甲床大致区域。
3. SAM 或轻量分割模型生成甲面 mask，允许前端人工点选修正。
4. 款式理解 Agent 抽取颜色、纹理、饰品、甲型、风格标签。
5. 局部 inpaint / ControlNet / IP-Adapter 只在 mask 内重绘，保留原图手纹、肤色、阴影和背景。
6. 质量评估：边界溢出、肤色漂移、光照一致、款式相似度、可商用自然度。

演示可以先 mock，但页面要显示“保留原图 + 只重绘甲面”的理念。

## 命题二：类 OpenClaw 的 Agent 运营客服

参考 OpenClaw 的“常驻、多渠道、可行动”思路，转成美团本地生活场景：

- 趋势 Agent：读取款式点击、收藏、订单、搜索词，发现本周爆款和滞销款。
- 运营 Agent：生成套餐、优惠、达人文案、门店海报文案和上新节奏。
- 客服 Agent：处理风格咨询、价格、预约、售后、退款、多轮追问。
- 执行 Agent：调用美团店铺 API 或 mock API，创建待确认任务，人工确认后执行。

## Demo 优先级

1. 先做前端可讲通的静态工作台。
2. 再接 DeerFlow 后端，把任务拆解和日志跑起来。
3. 视觉侧先做 mask 可视化和 mock 输出，最后接真实模型。
4. 运营侧先用 mock 数据生成日报和客服话术，最后接真实数据源。
